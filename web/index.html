💡 温馨提示：证据注册说明收起 ▲

本系统采用 路径引用 方式登记证据，不会复制或上传文件，仅在数据库中记录文件路径并计算哈希值用于完整性校验。

支持格式dd / img / e01 / vmdk / raw / dmp / pcap / pcapng / exe / dll / elf / jpg / png / gif / pdf / docx / xlsx / zip / tar / 7z / log / txt / csv 等
文件大小无限制。磁盘镜像和内存转储可达数百 GB，系统仅记录路径，不影响性能
路径要求必须是本地磁盘的 绝对路径（如 D:\cases\image.dd），不支持网络路径
证据名称建议格式：案件简称_证据类型_编号（如"入侵案_服务器镜像_001"），便于检索
哈希校验系统自动计算 SHA-256 值，确保证据完整性，后续每次分析前可验证文件未被篡改
安全提醒⚠️ 请确保证据文件路径正确且文件存在，否则后续分析任务将失败

📎 注册证据

证据注册是将本地文件路径登记到系统中，系统会自动计算 SHA-256 哈希值并记录审计日志。
适用于大文件（镜像、PCAP、内存转储等），无需上传，直接引用本地路径即可。

从下拉列表中选择取证工具后，系统会自动展示该工具的详细信息和推荐参数。以下是各工具的参数作用说明：

工具选择下拉列表标注了 ✅内置 或 🔧外部，选择后右侧自动显示工具详情和参数提示
证据关联必须选择已在当前案件中注册的证据，系统会自动将证据路径传给工具
参数(JSON)可选填写，用于指定工具的具体操作。留空则使用工具默认行为。格式为 JSON 键值对，如 {"action":"pcap_parse"}
常见参数action（指定操作类型）、mode（破解模式）、keyword（过滤关键字）、wordlist（字典路径）等
执行方式后台异步执行，提交后可前往"任务监控"页面查看进度和结果
结果获取任务完成后，在任务列表中点击"查看结果"即可看到分析数据，点击"生成报告"可导出格式化报告
